hexo搭建个人博客

  • 日志

第一次在github上搭建个人博客，在网上找了很多资料，都不是很详细，在这里记录一下搭建的过程。搭建资料1 搭建资料2

以下内容都是在windows环境下进行

1 .准备工作

搭建过程中需要用到的工具。 git、node、npm、hexo、github账号、域名（可选）。

系统环境：

  1. win7 64bit
  2. node 4.4.4
  3. git 2.8.2
  4. npm 2.15.1

2 .安装

安装git，安装教程

安装node，安装教程

hexo安装

Hexo安装

要用全局安装，加-g参数。
安装到哪个地方可以先到那个磁盘里新建一个文件夹，名字随意。
点击鼠标右键!

输入 npm install -g hexo

hexo会自动在目标文件夹下创建建立网站需要的所有文件。

然后初始化，执行init命令初始化hexo

输入 hexo init

到这里就可以在本地查看初步建立起来的博客，样子虽然不是很好看，但是后期可以调整。

输入 hexo g

生成静态文件

输入 hexo s

启动本地服务

在浏览器上输入localhost:4000 即可查看当前博客的样子了。

ctrl+c停止运行 继续接下来的操作

hexo目录就不写了，在搭建资料1有很详细的介绍
在目录下的source文件夹下的所有文件都是自己的，例如图片，CNAME之类的可以放入里面。

3.发布到github

登录github

有账号的可以忽略这步!

注册账号之后选择免费账户完成设置操作!

创建仓库
点击右上角加号，选择New repository!

这个仓库的名字需要和你的账号对应， 如 xxx.github.io
xxx就是你的账号
填入基本信息!

创建仓库完成之后我们需要一个SSH密匙来上传代码

右键 ls -al ~/.ssh

如果提示：No such file or directory 说明没有密匙

如果不存在没有关系，存在的话，可以直接在C盘.ssh文件夹下删除里面所有文件

输入 ssh-keygen -t rsa -C "xxx"

xxx是你的邮箱地址

其中需要输入3次回车，第一次可以直接回车，好像是创建的文件夹地址，
不用在意，第二次第三次是输入密码，该密码为上传代码时需要输入的密码，
如果不填直接回车的话提交代码时就没有密码提示

这样本机的密码就生成在C盘 用户下的.ssh文件夹下

打开id_rsa.pub文件 ctrl+a ctrl+c 全选并复制下来
如果看不到这个文件，你需要设置显示隐藏文件

然后打开github网站
在右上角选择Settings

选择 SSH and GPG keys

选择NEW SSH key

Title 随意填，Key值 就是id_rsa.pub下复制的信息

填写完毕之后点击Add SSH key 完成添加 SSH密匙操作

之后查看一下SSH是否设置成功 git@github.com的部分不要修改
输入 $ ssh -T git@github.com

如果是下面的反馈：

The authenticity of host ‘github.com (207.97.227.239)’ can’t be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

填入yes 回车

如果出现下面这条信息就是设置成功
Hi cnfeat! You’ve successfully authenticated, but GitHub does not provide shell access.

若有问题，请重新设置。常见错误请参考：

GitHub Help - Generating SSH Keys

GitHub Help - Error Permission denied (publickey)

设置用户信息
$ git config --global user.name

“cnfeat”//用户名

$ git config --global user.email

“cnfeat@gmail.com”//填写自己的邮箱

编辑_config.yml全局文件
在hexo文件夹中编辑找到_config.yml文件，请不要请用windows自带的编辑器

直接搜索 deploy 或者直接翻到最底部找到 deploy行

type：我这里使用的是git方式

repository： 填入的是github仓库中的ssh链接，直接粘贴复制即可

这里要注意的是type： repository： 冒号之后要打一个空格在输入内容。

4.发布到github

到这里我们需要把代码发布到github仓库中

在hexo文件夹下

输入 hexo g

静态化文件，静态化文件时会生成一个public的文件夹，该文件夹存放的是静态页面。

输入 hexo d

这样项目就发布到github中了。

进入github网站

点击Settings

找到GitHub Pages

Your site is published at 后面的网站部分就是生成的网站地址，可以直接点击查看该页面

5.设置域名

该步可自选是否操作。
查看页面时你会发现跟本地看到得不太一样，不要紧张F12查看会发现这里的原因是加载路径不对。

到阿里、腾讯等网站去购物一个域名。
购买完成后进去域名控制台，解析域名。
添加一个解析：
记录类型:A，主机记录:@,记录值:192.30.252.153 点击保存
在添加一个解析
记录类型:A，主机记录:@,记录值:192.30.252.154 点击保存
接着添加一个解析
记录类型:CNAME，主机记录:www,记录值:XXX.github.io 点击保存 XXX.github.io的值是创建仓库时填入的名称。
启动这些域名。

进入hexo文件夹，在source文件夹下新建一个文件文件，取名CNAME，全部大写，去掉.txt后缀,CNAME文件里只填写购买的域名地址。

然后在hexo目录下

输入命令 hexo g

输入命令 hexo d

打开浏览器，输入购买的域名地址。

如果连接超时，请等待一会，解析之后是需要一段时间反应的。
